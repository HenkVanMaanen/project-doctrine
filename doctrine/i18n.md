# Internationalization

## Standards

- ICU Message Format: https://unicode-org.github.io/icu/userguide/format_parse/messages/
- Unicode CLDR: https://cldr.unicode.org/

Applies to: webapp

## Requirements

- All user-facing strings MUST be externalized from day one.
- String keys MUST use a structured naming convention (e.g., `page.section.element`).
- ICU Message Format MUST be used for pluralization, gender, and variable interpolation.
- Date, time, number, and currency formatting MUST use locale-aware formatters.
- Text direction (LTR/RTL) MUST be supported in the layout.
- Language detection MUST follow: user preference > `Accept-Language` header > default locale.
- Translation files MUST be in a standard format (JSON, XLIFF, or PO).
- The default locale MUST be defined and all keys MUST have a default translation.

## See Also

- `accessibility.md` — language and text direction affect accessibility compliance

## Output Requirements

The generated i18n doc MUST:

- Specify the i18n framework/library for the chosen stack
- Define the string key naming convention with examples
- Define the translation workflow (extraction, translation, integration)
- List locale-sensitive formatting requirements
